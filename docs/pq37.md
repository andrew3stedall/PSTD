# PQ37 — bounded TCINFO root parsing

## Evidence reviewed

PQ36 corrected permutative decoding and exposed one genuine Heap-on-Node table context with `hidUserRoot=0x40`. It also left one decoded 208-byte payload at BID `0x74` structurally unsupported. The prior proposal suggested treating that payload as a possible row matrix, but the available evidence does not yet establish that relationship.

A Table Context root allocation begins with TCINFO. The row-data reference is an HNID union, so its low five bits determine whether it is a heap allocation ID or a node ID. Therefore BID `0x74` must not be correlated with row data merely because the values are numerically similar.

## Revised requirement

1. add a bounded parser for the 22-byte TCINFO header;
2. parse the four row-layout boundaries and require them to be monotonic;
3. parse `hidRowIndex`, `hnidRows`, and `hidIndex` without resolving them prematurely;
4. classify `hnidRows` as null, HID, or NID from its low five bits;
5. parse exactly `cCols` eight-byte TCOLDESC records with allocation bounds and basic bitmap/data-size validation; and
6. retain extraction behavior until the parser is wired to the real `hidUserRoot` allocation and the references are resolved through their correct domains.

## Implementation

`src/pst/tcinfo.rs` adds `TcInfo`, `TcColumnDescriptor`, and `HnidKind`. The parser is deliberately independent of BBT and heap traversal so malformed roots can be rejected before any reference resolution. Tests cover a valid bounded root, HNID union classification, truncated descriptor arrays, and non-monotonic region boundaries.
