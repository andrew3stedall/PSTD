# PRD Intake Template

Use this when submitting product ideas for planning.

## Problem

What problem are we solving?

## Users

Who is this for?

## Desired outcome

What should be true when this is complete?

## Draft scope

What do you think is in scope?

## Out of scope

What should not be included?

## Data notes

Sources, volume, quality, retention, or integration assumptions.

## UX notes

User flows, screens, CLI behaviour, API ergonomics, or documentation needs.

## Delivery constraints

Known technical, platform, timing, or operational limits.

## Success criteria

How will we know this has worked?

## Open questions

What decisions are still missing?
