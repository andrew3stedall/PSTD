## Evidence reviewed

## Revised requirement

Do not continue interpreting the current eight-byte table header as a confirmed real PST table descriptor. Add an explicit CI diagnostic that distinguishes:

1. descriptor fields genuinely present and suitable for bounds validation;
2. a contradictory parse where rows and values appear despite absent descriptor fields; and
3. insufficient signal requiring a broader fixture or different payload selection.

## Implementation

`scripts/pq32_descriptor_layout.py` reads the public fixture `run_summary.json`, writes `pq32_summary.json` and `pq32_summary.md`, and records a deterministic next blocker. CI runs it after the existing public PST progress report so every PQ run preserves the diagnosis as an artifact.
