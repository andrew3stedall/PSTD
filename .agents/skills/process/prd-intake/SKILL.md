# PRD Intake Process

## Purpose

Capture enough context to plan work without guessing.

## Required fields

- Problem.
- Users.
- Desired outcome.
- Draft scope.
- Out-of-scope items.
- Data notes.
- UX notes.
- Delivery constraints.
- Success criteria.
- Open questions.

## Output

Return a cleaned PRD summary and any missing decisions needed before issue planning.
