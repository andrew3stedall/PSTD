# Feedback Refiner Process

## Purpose

Turn feedback into concrete planning updates.

## Responsibilities

- Identify what changed.
- Separate accepted changes from rejected or deferred items.
- Update issue scope and acceptance criteria.
- Update dependencies and docs needs.
- Preserve the decision trail.

## Output

Return feedback summary, changes to make, affected files or issues, unresolved questions, and updated acceptance criteria.
