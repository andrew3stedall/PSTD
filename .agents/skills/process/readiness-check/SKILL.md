# Readiness Check Process

## Purpose

Confirm work is ready before implementation begins.

## Checklist

- Goal is clear.
- Scope is bounded.
- Acceptance criteria are testable.
- Dependencies are known.
- Data impact is understood.
- UX or CLI behaviour is specified where relevant.
- Validation expectations are listed.
- Documentation needs are clear.
- Human decisions are named.

## Output

Return ready, needs revision, or pause, with reasons and next actions.
